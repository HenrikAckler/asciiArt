# asciiArt
Converts images into ascii art. Currently functional but highly work in progress. Image below this readme generated with this program.

## Purpose
This program converts images to ASCII art using Java. It was created after looking into some of the readily available online solutions, and discovering that many of them took rather simple approaches, referencing a premade string of supposedly darker to lighter characters, comprising of only a small portion of the ascii space. This program is intended to explore new techniques for generating the list of ASCII characters by computing each character's luminosity, and investigating new methods for improving the generation for speed and/or quality.

## Design Goals
These are the following current design goals for this project:
 - Generates a list of luminosity sorted ASCII characters from scratch
   - Done!
 - Read in an image and map the brightness values
   - Done!
 - Reasonably performant, able to render simple images on a mid tier computer many times per second
   - Done!
 - Output smoothing of some sort, to prevent "dithering" due to excess repeated character changes
   - Algorithm in development
 - Video or Image output
   - To be completed
 - Simple command line interface
   - To be completed

## Optimization
It appears the primary cause of delay is getAsciiArt() in asciiGenerator.java, more specifically writing to the string. With my first test image, I was writing to the string twice per pixel, averaging ~3700ms to compute. Without writing to the string at all, this drops to a measly 16ms, enough to render 60 times per second. So I very much need to fix this. 

Implementing a StringBuilder instead pushes the performance down to the ~30ms range, over 2 orders of magnitude faster. My next hurdle was reading the images. I was previously reading the RGB value into a new Color object, then splitting that into RGB and averaging that. By instead using bitwise operators, I was able to greatly optimize the program, now running at around 15-16ms per frame.
